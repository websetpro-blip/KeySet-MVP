# План исследования: State Management для KeySet React App

## Цель
Выбрать оптимальное решение для управления состоянием веб-приложения KeySet (прототип вкладки "Парсинг")

## Структура данных приложения
- phrases: массив фраз (1000+ элементов)
- groups: дерево групп (вложенная структура)
- selectedGroup: текущая группа
- stopwords: массив минус-слов
- activityLog: массив логов
- undo/redo стек

## Сравниваемые решения
1. Context API (встроенный React)
2. Zustand
3. Jotai

## Критерии оценки
1. Производительность (частые обновления таблицы 1000+ строк)
2. Простота интеграции
3. DevTools поддержка
4. Размер bundle
5. Learning curve
6. Middleware (persist в localStorage)
